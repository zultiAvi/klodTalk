## Coder Output: Per-Session User List with Add/Remove and Scoped Broadcasting

### Changes Implemented

**server/session_manager.py:**
- Added `users: list` field to the `Session` dataclass (default: empty list)
- Updated `create_session()` to set `users=[user_name]` so the creator is the first registered user
- Updated `get_user_sessions()` to filter by `session.users` instead of project-level access
- Added `add_user_to_session()` method
- Added `remove_user_from_session()` method
- Added `migrate_sessions_add_users()` method for backward compatibility: populates empty `users` lists from project config at startup

**server/server.py:**
- Updated `_broadcast_to_session_users()` to use `session.users` instead of project users
- Updated `_mark_unread_for_others()` to use `session.users` instead of project users
- Updated `_notify_session_error()` to use `session.users` instead of project users
- Updated `_session_to_dict()` to include `users` field in the returned dictionary
- Updated permission checks in ALL handlers to use `session.users`:
  - `handle_stop()`
  - `handle_btw()`
  - `handle_close_session()`
  - `handle_delete_session()`
  - `handle_reopen_session()`
  - `handle_text()`
  - `handle_get_diff()`
  - `handle_revert_hunk()`
  - `handle_commit_and_push()`
- Kept project-level `users` check in `handle_new_session()` (session doesn't exist yet)
- Added `handle_add_user_to_session()` handler:
  - Only session owner (creator) can add users
  - Validates target user exists and has project access
  - Notifies both caller and target user
- Added `handle_remove_user_from_session()` handler:
  - Only session owner can remove users
  - Owner cannot remove themselves
  - Notifies both caller and target user
- Added message routing for `add_user_to_session` and `remove_user_from_session` types
- Added migration call at startup (`migrate_sessions_add_users`) after orphaned session cleanup

### Security Review Fixes

**CRITICAL-1 â€” Timing attack in verify_password (server/server.py):**
- Replaced `==` comparison with `hmac.compare_digest()` for constant-time hash comparison
- Added `import hmac` to top-level imports

**HIGH-1 â€” Removed user receives full users list (server/server.py):**
- Omitted `users` field from the notification sent to the evicted user in `handle_remove_user_from_session`; only the caller (session owner) sees the updated users list

**HIGH-2 â€” handle_mark_read has no session membership check (server/server.py):**
- Added session lookup and `user_name in session.users` membership check before calling `mark_read`; returns `Forbidden` error if unauthorized

**MEDIUM-1 â€” ValueError on double-remove (server/session_manager.py):**
- Wrapped `session.users.remove()` in try/except ValueError in `remove_user_from_session`

**MEDIUM-2 â€” Owner protection (server/session_manager.py):**
- Added owner guard in `remove_user_from_session()`: returns False if attempting to remove the session owner
- Added owner inclusion guard in `migrate_sessions_add_users()`: ensures `session.user_name` is in `session.users` after migration from project config
