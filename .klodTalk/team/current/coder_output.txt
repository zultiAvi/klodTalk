CODER OUTPUT ‚Äî Replace `disabled:` line flag with mandatory `## enabled` / `## disabled` H2 heading

## Summary
Replaced the freeform `disabled:` line flag mechanism with a mandatory H2 heading
(`## enabled` or `## disabled`) inside every team `.md` file. The heading is placed
between the team's one-line description and the `## Members` section. A team file
without one of these two headings is now treated as disabled and a warning is logged
by the server. No backward compatibility is preserved.

## Files changed

### Server
- /workspace/server/server.py
  - Rewrote `load_team()` (lines ~325-369). Removed all `disabled:` line parsing.
  - New scanner detects `## enabled` / `## disabled` via
    `stripped.lower() == "## enabled" / "## disabled"`. Heading lines are skipped
    (so they cannot be picked up as the description).
  - When neither heading is found, logs:
    `Team '<name>' missing mandatory '## enabled' or '## disabled' heading; treating as disabled`
    and returns `disabled=True`.
  - Returned dict shape unchanged: `{name, display_name, description, disabled}`.
  - `get_available_teams()` left untouched ‚Äî it already filters on `disabled`.
  - Session-start path (server.py:612) intentionally not touched per project instinct.
